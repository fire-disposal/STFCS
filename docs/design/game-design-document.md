## 一、游戏定位与愿景

### 1.1 游戏定义

**远行星号：太空战斗沙盒**（Starsector Tabletop Combat Sandbox）是一款**回合制太空舰船战斗桌面推演系统**，以虚拟桌面（VTT）形式呈现。

### 1.2 设计理念

| 维度 | 理念 |
|------|------|
| **核心体验** | 战术决策 > 操作技巧 |
| **节奏设计** | 深度思考 + 即时反馈 |
| **玩家角色** | 舰队指挥官，而非飞行员 |
| **社交属性** | 支持多人对战 + GM主持的战役模式 |

### 1.3 目标受众

- **战术游戏爱好者**：享受深度决策与战局推演
- **桌面战争游戏玩家**：熟悉回合制战术规则
- **远行星号粉丝**：了解原作世界观与舰船设定
- **小型社群**：2-6人在线对战或战役模式

### 1.4 独特卖点

| 特性 | 描述 |
|------|------|
| **确定性命中** | 基于几何验证而非概率随机，战术决策可精确预判 |
| **辐能资源管理** | 独特的"软/硬辐能"双轨系统，进攻与防御相互制约 |
| **护甲象限系统** | 六象限独立护甲，强调机动与攻击角度选择 |
| **护盾经济学** | 开盾消耗资源，关盾释放火力——持续权衡决策 |
| **三阶段移动** | 平移→转向→再平移，精确控制战场位置 |

## 二、核心游戏循环

### 2.1 单回合流程

```
回合开始
    │
    ├─► [部署阶段] 双方放置舰船 Token（无程序限制）
    │
    ├─► [玩家行动阶段]
    │       │
    │       ├─► 选择己方舰船
    │       │
    │       ├─► 平移阶段 A（可拆分）
    │       │       │
    │       │       ├─► 选择方向 → 锁定方向
    │       │       ├─► 移动部分距离
    │       │       ├─► [可选] 开火（射界实时更新）
    │       │       ├─► 继续移动（同一方向）
    │       │       ├─► [可选] 再次开火
    │       │       └─► 资源耗尽或手动结束
    │       │
    │       ├─► 转向阶段
    │       │       └─► 原地旋转（最大 Y 度）
    │       │
    │       ├─► 平移阶段 B（可拆分）
    │       │       └─► 流程同阶段 A，沿新朝向
    │       │
    │       ├─► 管理护盾（开启/关闭）
    │       ├─► 决定是否主动排散
    │       │
    │       └─► 约束条件：
    │           • 过载舰船无法行动
    │           • 开火后禁止本回合主动排散
    │           • 每武器每回合限射一次
    │           • 移动拆分时方向锁定，不可切换
    │
    ├─► [敌方行动阶段] GM 操作敌方舰船，流程同上
    │
    ├─► [回合结算] GM 按下"回合结束"
    │       │
    │       ├─► 辐能排散：Flux_Soft 减少 Flux_Dissipation
    │       ├─► 过载恢复：Flux_Total = Flux_Max / 2
    │       ├─► 主动排散完成：Flux 全清空
    │       ├─► 武器冷却递减（跨回合）
    │       ├─► 弹药重装（回合结束自动）
    │       │
    │       └─► 进入下一回合
    │
    └─► [战局判断] 
        • 一方舰船全部摧毁 → 战斗结束
        • 或 GM 设定的其他胜利条件
```

### 2.2 战役循环（可选）

```
战役开始
    │
    ├─► 设定战役背景与目标
    ├─► 玩家组建舰队（舰船自定义系统）
    ├─► 序列战斗场景
    │       │
    │       ├─► 场景 1: 部署 → 战斗 → 结算
    │       ├─► 场景 2: 残局继续或新部署
    │       └─► ...
    │
    ├─► 战役结算（资源/损伤统计）
    └─► 剧情推进或战役结束
```

## 三、舰船系统设计

### 3.1 舰船 Token 属性

舰船是游戏的核心单位，每个 Token 代表一艘独立的战斗舰船。

#### 基础生存属性

| 属性 | 定义 | 玩法意义 |
|------|------|----------|
| **Hull（结构值）** | 舰船生命值 | 降至0即摧毁，核心生存指标 |
| **Armor（护甲）** | 六象限独立护甲值 | 局部损伤，强调攻击角度选择 |

#### 辐能属性（核心资源系统）

| 属性 | 定义 | 玩法意义 |
|------|------|----------|
| **Flux_Max** | 辐能容量上限 | 过载阈值，资源上限 |
| **Flux_Soft** | 软辐能 | 可自然排散，来自护盾维持与武器射击 |
| **Flux_Hard** | 硬辐能 | 无法自然排散，来自护盾吸收伤害 |
| **Flux_Dissipation** | 每回合排散量 | 决定恢复速度，关键战术节奏指标 |

#### 机动属性

| 属性 | 定义 | 玩法意义 |
|------|------|----------|
| **Speed_X** | 基础航速 | 每阶段移动单位数 |
| **Turn_Rate_Y** | 每回合最大转向角度 | 控制机动灵活性 |
| **Heading** | 当前船头朝向 | 决定移动方向与射界基准 |

#### 护盾属性

| 属性 | 定义 | 玩法意义 |
|------|------|----------|
| **Shield_Type** | 前盾/全盾 | 前盾固定中心点，全盾可自由调整 |
| **Shield_Arc** | 护盾覆盖角度 | 决定防护范围 |
| **Shield_Efficiency** | 吸收倍率 | 伤害→硬辐能转化率 |
| **Shield_Up_Cost** | 每回合软辐能消耗 | 开盾的"租金"成本 |

#### 武器挂载点

| 挂载点属性 | 定义 | 玩法意义 |
|-----------|------|----------|
| **位置** | 相对舰船中心坐标 | 决定武器射击起点 |
| **朝向** | 基准射击方向 | 射界扇形中心轴 |
| **射界 Arc** | 可射击角度范围 | 决定火力覆盖范围 |
| **尺寸 Size** | SMALL/MEDIUM/LARGE | 武器尺寸兼容限制 |
| **类型 Type** | 挂载点（固定）/ 炮塔（旋转） | 挂载点±10°，炮塔自定义 |

### 3.2 舰船尺寸分级

| 尺寸 | 特性定位 | 典型 OP 容量 | 典型角色 |
|------|----------|-------------|----------|
| **护卫舰 FRIGATE** | 高机动、低生存 | 5-10 | 侦察、骚扰、侧翼突击 |
| **驱逐舰 DESTROYER** | 平衡型 | 10-20 | 通用战斗、护航 |
| **巡洋舰 CRUISER** | 重火力、中机动 | 20-40 | 主力输出、指挥舰 |
| **主力舰 CAPITAL** | 极高火力、低机动 | 40-80 | 战场核心、旗舰 |

### 3.3 舰船自定义系统

#### 装备点数（OP）限制

每艘舰船拥有**装备点数容量**，限制武器配置总量：
- 小型武器 OP ≈ 2-5
- 中型武器 OP ≈ 5-10
- 大型武器 OP ≈ 10-20

**软限制设计**：允许超出 OP 配置，但标记为"不合规"，GM 可选择是否允许。

#### 武器配置流程

```
选择舰船型号
    │
    ├─► 查看挂载点列表（位置、尺寸、射界）
    │
    ├─► 为每个挂载点选择武器
    │       │
    │       ├─► 尺寸兼容验证（大型槽可装小型武器）
    │       ├─► 类型限制验证（部分槽位限制武器类别）
    │       ├─► OP 预算计算
    │       │
    │       └─► 内置武器（不可更换）直接锁定
    │
    ├─► 查看装备摘要
    │       • 总伤害输出
    │       • 总辐能消耗
    │       • 有效射程范围
    │       • OP 使用率
    │
    └─► 保存为"舰船变体"
        • 预设变体（系统提供）
        • 自定义变体（玩家创建）
```

## 四、移动系统设计

### 4.1 空间模型

- **二维平面坐标系**：像素或自定义单位，非方格限制
- **连续空间**：舰船可在任意位置移动，无格子约束
- **朝向重要**：船头方向决定移动基准轴与射界基准

### 4.2 三阶段移动系统

**设计目的**：将移动分解为可控的三个阶段，支持精细战术操作。

```
阶段 A：平移
    │
    ├─► 前进/后退（最大 2X 单位）
    │       └─► 沿当前 Heading 方向
    │
    ├─► 横移（最大 X 单位）
    │       └─► 沿中轴线切线方向（左/右）
    │
    └─► 约束：不能同时前进后退和横移

阶段 B：转向
    │
    ├─► 原地旋转（最大 Y 度）
    │
    └─► 约束：原地旋转，不改变位置

阶段 C：再平移
    │
    ├─► 前进/后退（最大 2X 单位）
    │       └─► 沿**新 Heading** 方向
    │
    ├─► 横移（最大 X 单位）
    │       └─► 沿**新中轴线**切线方向
    │
    └─► 约束：同样不能同时进行
```

**战术意义**：
- 先移后转：调整射击角度
- 先转后移：改变前进方向
- 两次平移：实现"斜线机动"

### 4.3 移动拆分机制

**核心规则**：在同一平移阶段内，若选择了单一移动方向，可将该方向的移动资源拆分为任意多次执行，中间可穿插其他行动。

```
移动拆分示例（平移阶段 A）

场景：舰船 Speed_X = 10，选择"向左横移"方向（最大 X = 10 单位）

传统执行：
    └─► 向左移动 10 单位 → 阶段结束

拆分执行（允许）：
    │
    ├─► 向左移动 4 单位
    │       └─► 消耗 4/10 资源
    │
    ├─► 右舷武器组开火（射界现在覆盖目标）
    │
    ├─► 向左移动 3 单位
    │       └─► 消耗 7/10 资源
    │
    ├─► 左舷武器组开火（新位置射界覆盖另一目标）
    │
    ├─► 向左移动 3 单位
    │       └─► 消耗 10/10 资源
    │
    └─► 平移阶段 A 结束
```

**关键约束**：

| 约束 | 说明 |
|------|------|
| **方向锁定** | 拆分后必须保持同一方向，不能中途切换方向 |
| **资源总量** | 所有拆分移动的总距离不超过该阶段最大值 |
| **行动穿插** | 仅允许在移动拆分点进行开火行动，不允许穿插转向 |
| **阶段边界** | 拆分只在同一平移阶段内有效，跨阶段不可延续 |

**战术意义**：

| 战术应用 | 示例 |
|----------|------|
| **动态射击窗口** | 移动过程中寻找最佳射界覆盖时机开火 |
| **多目标切换** | 左移→右舷开火→继续左移→左舷开火，同时攻击两侧目标 |
| **渐进逼近** | 分步靠近目标，每步开火后继续逼近 |
| **射击节奏控制** | 避免一次性移动后才发现射界不理想 |

**交互设计要点**：

```
平移阶段交互流程（拆分模式）

进入平移阶段
    │
    ├─► 选择移动方向（前/后/左/右）
    │       └─► 方向锁定，显示可用资源总量
    │
    ├─► 执行移动（第一次）
    │       │
    │       ├─► 拖拽或输入距离
    │       ├─► 显示剩余资源
    │       │
    │       └─► 移动后状态：
    │           • 位置更新
    │           • 射界实时更新
    │           • 可选择：继续移动 / 开火 / 结束阶段
    │
    ├─► [可选] 开火行动
    │       └─► 选择武器 → 选择目标 → 开火
    │       └─► 开火后仍可继续移动（方向已锁定）
    │
    ├─► 执行移动（第二次...）
    │       └─► 重复上述流程
    │
    └─► 结束平移阶段
            ├─► 手动点击"结束阶段"
            ├─► 或资源耗尽自动结束
            └─► 进入下一阶段（转向/再平移）
```

**重要说明**：

- 拆分机制仅在**单一方向选择**后生效
- 若玩家在阶段开始时未明确方向（如犹豫状态），首次移动即锁定方向
- 开火行动不消耗移动资源，但受辐能/冷却等限制
- 过载舰船无法执行任何移动或开火行动

### 4.4 移动决策要素

| 决策点 | 考虑因素 |
|------|----------|
| **前进距离** | 拉近射程/保持距离/撤退 |
| **转向角度** | 调整射界覆盖/暴露薄弱护甲象限/规避敌方火力 |
| **横移方向** | 侧向机动/保持队形/躲避追击 |
| **过载状态** | 过载舰船完全无法移动 |

## 五、战斗系统设计

### 5.1 武器系统

#### 武器基础属性

| 属性 | 定义 | 玩法意义 |
|------|------|----------|
| **伤害类型** | 动能/高爆/破片/能量 | 不同伤害修正系数 |
| **伤害值** | 单次射击伤害 | 基础火力指标 |
| **射程** | 最大射击距离 | 战术站位依据 |
| **射界 Arc** | 可射击角度范围 | 决定火力覆盖方向 |
| **冷却时间** | 射击后冷却回合数 | 火力节奏控制 |
| **辐能消耗** | 射击产生的软辐能 | 资源消耗成本 |
| **弹药** | 有限弹药或无限 | 限制连续射击 |
| **点防御 PD** | 是否可拦截导弹/战机 | 防御角色 |

#### 武器伤害类型修正

| 伤害类型 | 对护盾倍率 | 对护甲穿甲强度 | 对船体倍率 | 战术定位 |
|---------|-----------|---------------|-----------|----------|
| **动能 KIN** | 2.0× | 0.5× | 1.0× | 反盾专用，快速削盾 |
| **高爆 HE** | 0.5× | 2.0× | 1.0× | 反甲专用，穿透护甲 |
| **破片 FRAG** | 0.25× | 0.25× | 1.0× | 反船体，对防护无效 |
| **能量 NRG** | 1.0× | 1.0× | 1.0× | 万金油，无修正 |

**战术组合意义**：
- 动能武器削盾 → 高爆武器穿透护甲 → 破片武器收割船体
- 能量武器独立有效，适合单舰作战

### 5.2 射界判定系统

#### 射界几何

```
                    武器位置
                        │
        ┌───────────────┼───────────────┐
        │               │               │
        │    射界扇形    │    射界扇形    │
        │   (左边界)     │   (右边界)     │
        │               │               │
        └─◄─────────────┼─────────────►─┘
                        │
                   射界中心轴
                   (武器朝向)
                        │
                        ▼
                   最大射程边界
```

#### 射界验证流程

```
测距
    │
    ├─► 计算武器位置与目标中心距离
    │
    ├─► 对比武器射程
    │       └─► 超出射程 → 无效
    │
    ├─► 计算相对角度
    │       │
    │       ├─► 目标相对于武器位置的角度
    │       ├─► 武器当前朝向（舰船 Heading + 挂载点 Facing）
    │       │
    │       └─► 角度偏差 > Arc/2 → 无效
    │
    └─► 全部通过 → 目标可攻击
```

#### 炮塔 vs 挂载点

| 类型 | 射界限制 | 玩法意义 |
|------|----------|----------|
| **挂载点 Mount** | 固定±10° | 需要舰船转向来调整射界 |
| **炮塔 Turret** | 自定义弧度（如90°、180°） | 独立射界，更灵活 |

### 5.3 伤害结算流程

```
命中判定（确定性）
    │
    ├─► 射界验证 → 不在射界 = 无效
    ├─► 射程验证 → 超出射程 = 无效
    │
    └─► 全部通过 → 确定命中（无随机因素）

护盾阶段
    │
    ├─► 判断目标护盾状态
    │       │
    │       ├─► 护盾关闭 → 跳过护盾阶段
    │       ├─► 护盾开启 → 判断命中位置是否在护盾覆盖范围内
    │       │
    │       └─► 命中护盾 → 硬辐能 = 伤害 × Shield_Efficiency × 伤害类型倍率
    │               └─► 硬辐能无法自然排散
    │
    └─► 未命中护盾 → 进入护甲阶段

护甲阶段
    │
    ├─► 选择攻击象限（6象限系统）
    │       │
    │       ├─► 象限 1: 0°-60°（船头右前方）
    │       ├─► 象限 2: 60°-120°（右侧）
    │       ├─► 象限 3: 120°-180°（右后方）
    │       ├─► 象限 4: 180°-240°（左后方）
    │       ├─► 象限 5: 240°-300°（左侧）
    │       └─► 象限 6: 300°-360°（左前方）
    │
    ├─► 计算有效护甲
    │       │
    │       ├─► C = max(当前象限护甲, 最大护甲 × 最小减免比例)
    │       ├─► 穿甲强度 Z = 伤害 × 穿甲系数（按伤害类型）
    │       │
    │       └─► 有效伤害 D = 伤害 × (Z / (Z + C))
    │               └─► 高护甲 → 低伤害
    │               └─► 低护甲 → 高伤害
    │
    ├─► 减伤上限检查
    │       └─► 若减伤比例超过上限，强制最低伤害
    │
    ├─► 应用护甲损伤
    │       └─► 当前象限护甲减少（局部损伤）
    │
    └─► 剩余伤害 → 应用至船体

船体阶段
    │
    ├─► 应用船体伤害
    │       └─► Hull_Current 减少
    │
    └─► Hull_Current ≤ 0 → 舰船摧毁
```

### 5.4 六象限护甲系统

**设计意义**：
- 局部损伤累积 → "装甲打穿"效果
- 机动选择暴露哪一象限 → 主动防御决策
- 高爆武器集中攻击同一象限 → 突破策略

## 六、辐能系统设计

### 6.1 辐能类型

| 类型 | 来源 | 排散方式 | 玩法意义 |
|------|------|----------|----------|
| **软辐能 Soft Flux** | 武器射击、护盾维持 | 每回合自然排散 | 可恢复资源 |
| **硬辐能 Hard Flux** | 护盾吸收伤害 | 无法自然排散，需主动排散或关盾 | 持久压力 |

### 6.2 辐能资源循环

```
辐能产生
    │
    ├─► 武器射击 → 软辐能 +Flux_Weapon
    ├─► 护盾维持 → 软辐能 +Shield_Up_Cost/回合
    ├─► 护盾吸收伤害 → 硬辐能 +伤害×Efficiency
    │
    └─► 总辐能 Flux_Total = Flux_Soft + Flux_Hard

辐能排散
    │
    ├─► 自然排散（回合结算）
    │       └─► Flux_Soft 减少 Flux_Dissipation
    │       └─► 开盾状态下仍可排散软辐能
    │
    ├─► 主动排散（玩家选择）
    │       └─► 本回合放弃所有行动
    │       └─► 禁止开盾、禁止开火
    │       └─► Flux_Soft = 0, Flux_Hard = 0
    │
    └─► 过载恢复（回合结算）
            └─► Flux_Total = Flux_Max / 2
```

### 6.3 过载机制

```
过载触发
    │
    └─► Flux_Total ≥ Flux_Max → 过载

过载效果
    │
    ├─► 无法移动
    ├─► 无法开火
    ├─► 护盾强制关闭
    ├─► 完全暴露状态
    │
    └─► 持续至本回合结束

过载恢复
    │
    └─► 下回合结束时
            └─► Flux_Total = Flux_Max / 2
            └─► 恢复行动能力
            └─► 护盾可重新开启
```

### 6.4 辐能策略决策

| 决策点 | 选项与权衡 |
|------|----------|
| **开盾时机** | 开盾吸收伤害但产生硬辐能；关盾保持辐能空间但暴露 |
| **射击频率** | 射击产生软辐能，辐能接近上限时需谨慎 |
| **主动排散** | 放弃本回合行动换取辐能清空——进攻间隙的恢复窗口 |
| **过载风险** | 过载=完全暴露——高风险时刻，需提前规划避免 |

## 七、护盾系统设计

### 7.1 护盾类型

| 类型 | 特性 | 玩法意义 |
|------|------|----------|
| **前盾 Front Shield** | 固定中心点，覆盖前方角度 | 需要转向调整防护方向 |
| **全盾 Omni Shield** | 可自由调整中心点 | 独立防护方向，更灵活 |

### 7.2 护盾经济学

```
护盾开启成本
    │
    ├─► 每回合软辐能消耗 Shield_Up_Cost
    │       └─► 持续资源消耗（"租金")
    │
    └─► 吸收伤害 → 硬辐能积累
            └─► 长期压力（"债务"）

护盾关闭收益
    │
    ├─► 停止软辐能消耗
    ├─► 硬辐能可通过主动排散清除
    │
    └─► 但——完全暴露，无防护

护盾策略循环
    │
    ├─► 高威胁阶段 → 开盾吸收伤害
    │       └─► 辐能积累，但保护船体/护甲
    │
    ├─► 低威胁阶段 → 关盾恢复
    │       └─► 主动排散清除硬辐能
    │
    └─► 循环往复，攻防节奏切换
```

### 7.3 护盾覆盖判定

```
护盾覆盖区域
    │
    ├─► 以 Shield_Orientation 为中心轴
    ├─► 覆盖 Shield_Arc 角度范围
    │       └─► 如 Shield_Arc = 90° → 前方±45°
    │
    └─► 护盾半径 Shield_Radius
            └─► 决定护盾"大小"

命中判定
    │
    ├─► 计算命中位置相对于舰船中心的角度
    │
    └─► 判断是否在护盾覆盖角度范围内
            └─► 在范围内 → 护盾吸收
            └─► 不在范围内 → 直接击中护甲
```

## 八、回合制简化原则

### 8.1 与实时游戏的对比

| 概念 | 实时游戏 | 回合制简化 |
|------|----------|------------|
| **命中判定** | 弹道追踪 + 概率命中 | **确定性射界验证**（即时命中） |
| **射弹飞行** | 物理弹道计算 | **不存在**（伤害即时应用） |
| **冷却机制** | 实时倒计时 | **跨回合冷却**（回合结束递减） |
| **弹药重装** | 实时装填动画 | **回合结束自动重装** |
| **多武器射击** | 连续开火 | **每回合每武器限射一次** |
| **辐能管理** | 实时消耗/排散 | **回合结算辐能变化** |

### 8.2 核心简化理念

**确定性原则**：
- 所有伤害基于确定性几何验证（射界+射程）
- 无随机命中概率，战术决策可精确预判结果
- 玩家可完全计算攻击效果，决策有据可依

**回合边界原则**：
- 所有持续效果以回合为边界结算
- 冷却、弹药、辐能排散在回合结束时统一处理
- 玩家每回合只需做决策，无需实时操作

## 九、交互设计

### 9.1 武器开火交互流程

```
正常状态（IDLE）
    │
    ├─► 选中舰船 → 显示所有武器射界叠加
    │
    ├─► 点击武器按钮 → 进入瞄准模式（TARGETING）
    │       │
    │       ├─► 高亮选中武器射界（蓝色）
    │       ├─► 高亮可攻击目标（绿色光环）
    │       ├─► 不可攻击目标灰色标记
    │       │
    │       └─► 约束：
    │           • 武器冷却中 → 禁用
    │           • 本回合已射击 → 禁用
    │           • 舰船过载 → 全武器禁用
    │
    ├─► 点击可攻击目标 → 发送开火命令 → 返回 IDLE
    │       └─► 显示伤害效果反馈
    │
    ├─► 点击不可攻击目标 → Toast 提示无效原因
    │
    └─► 点击空白/ESC/取消 → 退出瞄准模式
```

### 9.2 状态反馈设计

#### 武器按钮状态

| 状态 | 视觉反馈 |
|------|----------|
| **就绪** | 绿色边框，可点击 |
| **冷却中** | 黄色边框，显示冷却倒计时，禁用 |
| **弹药耗尽** | 红色边框，显示"空(0)"，禁用 |
| **本回合已射击** | 灰色背景，显示"已射击"，禁用 |
| **选中状态** | 蓝色高亮边框，蓝色背景 |

#### 目标舰船状态

| 状态 | 视觉反馈 |
|------|----------|
| **可攻击** | 绿色光环，可点击样式 |
| **不可攻击** | 灰色×标记，禁用样式 |
| **威胁等级** | 颜色编码（红=高、黄=中、绿=低） |

#### 射界可视化层次

| 层级 | 内容 | 颜色 |
|------|------|------|
| **IDLE 模式** | 所有武器射界叠加 | 橙色半透明 |
| **TARGETING 模式** | 仅选中武器射界高亮 | 亮蓝色，射程圈虚线 |

### 9.3 移动交互设计

```
移动阶段选择
    │
    ├─► 点击舰船 → 显示移动面板
    │       │
    │       ├─► [平移阶段 A] 按钮
    │       ├─► [转向阶段] 按钮
    │       ├─► [平移阶段 B] 按钮
    │       │
    │       └─► 每阶段完成后自动切换到下一阶段

平移阶段交互（支持拆分）
    │
    ├─► 进入平移阶段
    │       │
    │       ├─► 显示可用资源总量（如"可用: 10 单位"）
    │       ├─► 方向选择按钮：[前进] [后退] [左移] [右移]
    │       │
    │       └─► 选择方向后：
    │           • 方向锁定（高亮显示）
    │           • 显示该方向最大移动范围指示器
    │
    ├─► 执行移动（拆分模式）
    │       │
    │       ├─► 输入或拖拽确定本次移动距离
    │       │       └─► 约束：不超过剩余资源
    │       │
    │       ├─► 移动执行后：
    │       │       ├─► 舰船位置更新
    │       │       ├─► 射界实时更新显示
    │       │       ├─► 剩余资源显示（如"剩余: 6 单位"）
    │       │       │
    │       │       └─► 可选操作：
    │       │           ├─► [继续移动] → 保持方向，继续拆分
    │       │           ├─► [开火] → 进入瞄准模式
    │       │           ├─► [结束阶段] → 进入下一阶段
    │       │
    │       ├─► [开火后状态]
    │       │       ├─► 武器进入冷却/已射击状态
    │       │       ├─► 辐能增加
    │       │       ├─► 仍可继续移动（方向已锁定）
    │       │       │
    │       │       └─► 返回移动选择界面
    │       │
    │       └─► 资源耗尽自动结束阶段
    │           └─► 或手动点击 [结束阶段]

转向阶段交互
    │
    ├─► 显示当前朝向指示器
    ├─► 显示可旋转范围（最大 Y 度）
    ├─► 拖拽旋转船头方向
    ├─► 或输入精确角度
    │
    └─► 约束可视化：超出角度显示警告

移动预览与确认
    │
    ├─► 显示移动轨迹预览线（拆分时显示多段）
    ├─► 显示转向前后对比
    ├─► 实时显示射界变化（移动过程中动态更新）
    │
    └─► 每次移动/开火操作即时生效（无撤销，除非 GM 允许）
```

**拆分移动 UI 元素**：

| 元素 | 设计 |
|------|------|
| **资源条** | 显示已用/总量，如 `███████░░░ 7/10` |
| **方向锁定指示** | 高亮选中方向按钮，其他方向灰色禁用 |
| **射界实时更新** | 移动过程中射界扇形跟随舰船位置实时变化 |
| **继续移动按钮** | 仅在有剩余资源时可用 |
| **开火按钮** | 仅在有未射击武器且辐能允许时可用 |
| **结束阶段按钮** | 随时可点击，提前结束当前阶段 |
