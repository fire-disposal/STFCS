Token（舰船）需要有一条中心点和中轴线，规定船头和船尾。
舰船移动逻辑，以下三个步骤必须按照顺序进行，舰船具有航速2X，最大转向角度Y
沿着船头朝向前进/后退最多2X的距离 或 沿着中轴线切线的方向横移X的距离→转向最多Y度→沿着船头朝向前进/后退最多2X的距离 或 沿着中轴线切线的方向横移X的距离

辐能值（Flux）一个具有最大值的量表，最大值由舰船的固有属性“辐能容量”决定（可自定义），每回合都会下降等同于舰船的固有属性“辐散”（可自定义）的数值
武器具有“辐能产生”属性，决定了每次开火舰船自身会增加的辐能值。被启动的护盾也会根据自身的“护盾维持消耗”属性产生辐能，护盾防御住武器攻击也会产生相当于武器伤害*护盾效率的辐能。
护盾防御武器攻击产生的辐能为“硬辐能”，这一部分在护盾关闭前无法通过“辐散”降低，但武器开火和护盾维持产生的辐能为“软辐能”，其可以在护盾开启时降低。
当舰船的辐能到达上限时，舰船会立刻“过载”，直到其下个回合结束，所有武器不可用，护盾不可用，并且无法移动，过载结束时，辐能降低到最大值的一半
舰船可以在自己的回合“主动排散”，关闭自身护盾，禁用武器直到自身回合结束，并清空自身的辐能

Token（舰船）需要有以下这些数据，且这些数据全部都应可自定义
结构值（一个具有最大值的量表，代表Token的生命值）
护甲值（分六个象限各自独立的护甲值，其越高就代表该区域抵抗伤害的能力越强）
辐能值（一个具有最大值的量表，代表舰船的战斗能量资源）
航速（舰船单阶段内可以行进的距离）
转向速度（舰船单回合内可以转向的最大角度）
挂载点/炮塔位（分大中小尺寸，大型武器点位可以向下兼容安装小型武器，决定了武器可以指向的角度，挂载点上的武器只能指向其朝向左右各10°的范围，炮塔的指向范围则可以自定义）
插件栏位（可以自行填写插件名称及描述的自定义记录栏位）
护盾类型（护盾具有展开的中心点，前盾舰船的展开中心点只能是舰船正前方，全盾舰船的中心点则不做限制）
护盾半径（护盾全部为以舰船中心点为圆心的圆形，这标志着舰船护盾的半径，护盾展开中心点也只能位于这个圆上）
护盾效率（决定了护盾吸收伤害时会产生多少硬辐能的倍率，其公式为辐能=吸收伤害×护盾效率）
最大护甲减伤比，最小护甲减伤比
射程比率（武器的真实射程为武器面板射程×射程比率）

护甲与伤害的机制
设有最大护甲值A，当前护甲值为B，则有计算护甲值C
设有单发攻击伤害X，则有穿甲强度Z=X，有对护盾伤害Y=X
伤害类型分为高爆，动能，破片，能量，其会对上述数据做出如下修正
高爆：穿甲强度Z=X×2，对护盾伤害Y=X×0.5
动能：穿甲强度Z=X×0.5，对护盾伤害Y=X×2
破片：穿甲强度Z=X×0.25，对护盾伤害Y=X×0.25
能量：对上述数据不做修正。

若攻击命中舰船护盾，则对舰船造成Y×A（对护盾伤害×护盾效率）点硬辐能
若攻击命中护甲，每一发攻击命中时，有以下公式
最终伤害D=X×Z / ( Z+C )，其中C=B或者A×最小护甲减伤比，取其中较大者
而(X-D)/X若大于最大护甲减伤比，则D=X×（1-最大护甲减伤比）
