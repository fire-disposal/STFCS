# 桌游设计师视角的 UX 批判

## 核心问题

**当前星图是一个控制面板，但桌游需要的是一个讨论画布。**

```
当前设计：
DM 点击节点 → 立即航行
            ↳ 其他玩家只能在旁边看

桌游需要：
玩家甲：「去矿区看看？」
玩家乙：「星云里可能有物资。」
玩家丙：「投票吧。」
DM：       「好，出发。」→ DM 执行
```

## 星图的三层用途

| 层       | 谁     | 做什么                   | 当前支持          |
| -------- | ------ | ------------------------ | ----------------- |
| **浏览** | 所有人 | 看节点分布、名称、状态   | ✅ 可视化         |
| **讨论** | 所有人 | 指向节点、表达意向、辩论 | ❌ 只有 GM 可交互 |
| **执行** | DM     | 确认群体决策、触发航行   | ✅ 但缺确认步骤   |

## 具体问题

1. **DM 孤独决策** — 点击即航行，没有讨论空间
2. **玩家无参与感** — 只能看，不能"指"
3. **没有"指向"机制** — 无法在不触发行动的情况下标记节点
4. **航行是黑盒** — 点击后直接出结果，没有"准备出发"的仪式感

## 改进方案

### 交互流程重设计

```
┌─ 浏览阶段 ─────────────────────────┐
│ 所有人可点击节点 → 节点高亮（指向）  │
│ 多次点击切换指向，不触发任何行动      │
└─────────────────────────────────────┘
          ↓ 讨论、投票、决策
┌─ 确认阶段（仅 DM）─────────────────┐
│ DM 面板显示「当前指向: 矿区」        │
│ DM 点击 [出发] 按钮 → 执行航行       │
│ 或 DM 点击不同节点改变指向           │
└─────────────────────────────────────┘
          ↓ 航行执行
┌─ 航行阶段 ─────────────────────────┐
│ 动画反馈 + 通知                     │
└─────────────────────────────────────┘
```

### 具体改动

1. **所有人可点击节点** — 点击只"高亮"，不触发 travel
2. **高亮状态同步** — 用 uiStore 或本地状态记录 `highlightedNodeId`
3. **DM 面板 [出发] 按钮** — 仅在 DM 且 WORLD 模式且选中节点时显示
4. **玩家看到高亮** — 所有客户端看到同一高亮节点

### 需改动的文件

| 文件                      | 改动                                                   |
| ------------------------- | ------------------------------------------------------ |
| StarMapRenderer.tsx       | 移除 onPointertap 中的 travel 调用，改为 emit 高亮事件 |
| WorldInfoPanel.tsx        | 高亮节点显示[出发]按钮，点击后调用 travel              |
| DMControlSidebarPanel.tsx | WORLD 模式下显示当前高亮 + [出发]                      |
